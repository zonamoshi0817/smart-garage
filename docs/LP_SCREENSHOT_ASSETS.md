## 1. ヒーロー用メインスクショ

### ファイル名
`/public/lp-screenshots/hero-maintenance-focus.png`

### 推奨サイズ
- **幅**: 1200px
- **高さ**: 900px
- **アスペクト比**: 4:3

### 要件
- **構図**: ダッシュボードの左ナビを切り、中心の「次回メンテ提案」周辺をズーム
- **強調要素**: 次回メンテ提案カードを中心に配置

  - プラン表記: `Premium プラン`
  - 金額: `¥xx,xxx` などのプレースホルダー
- **視覚効果**: 強調したい要素以外は軽くぼかす or 薄くする（スポットライト効果）
- **背景**: グレー系（`bg-gray-50`相当）で統一

### デザインガイド
- ブラウザウィンドウのヘッダー（赤/黄/緑のボタン）を含める
- サイドバーは表示しない（または薄くぼかす）
- メインコンテンツエリアのみを大きく表示
- 「次回メンテ：残 1,234km」のカードが目立つように

## 3. 画像形式・品質

### 形式
- **PNG** (透過が必要な場合) or **WebP** (推奨、ファイルサイズ最適化)
- **JPG** も可（透過不要の場合）

### 品質設定
- **圧縮率**: 80-85% (WebP/JPG)
- **最適化**: Next.js Image Optimization を使用

### ファイルサイズ目標
- ヒーロー用: **200KB以下**
- 機能別: **150KB以下** (各画像)

---

## 4. 実装時の注意事項

### Next.js Image コンポーネント使用

```tsx
<Image
  src="/lp-screenshots/hero-maintenance-focus.png"
  alt="GarageLog - 次回メンテ提案機能"
  width={1200}
  height={900}
  className="w-full h-full object-cover"
  priority  // ヒーロー用のみ
  placeholder="blur"
  blurDataURL="..." // プレースホルダー用の小さな画像
/>
```

### CLS対策
- **width/height**: 必ず指定（アスペクト比維持）
- **aspect-ratio**: CSS側でも指定（念のため）
- **placeholder="blur"**: 読み込み中のちらつき防止

### レスポンシブ対応
- **srcset**: Next.jsが自動生成
- **sizes**: `(max-width: 768px) 100vw, 50vw` など

## 5. デザインツール推奨設定

### Figma / Sketch
- **エクスポート**: PNG 24bit または WebP
- **リサイズ**: 上記推奨サイズに合わせる
- **ぼかし効果**: 背景に `blur(8px)` 程度

### 撮影ツール（実際のアプリから）
- **ブラウザ拡張**: Full Page Screen Capture
- **設定**: 1200px幅で撮影 → 必要に応じてトリミング

## 7. チェックリスト

画像を追加する前に以下を確認：

- [ ] ファイル名が正しい（ケースセンシティブ）
- [ ] `/public/lp-screenshots/` ディレクトリに配置
- [ ] 推奨サイズに合わせている
- [ ] ファイルサイズが目標内
- [ ] デモデータが適切（個人情報なし）
- [ ] 強調要素が明確
- [ ] CLS対策済み（width/height指定）
- [ ] アクセシビリティ（alt属性適切）

---

## 8. 画像差し替え手順

1. `/public/lp-screenshots/` ディレクトリに画像を配置
2. `src/app/(marketing)/page.tsx` の該当箇所のコメントアウトを外す
3. 画像パスを確認（`/lp-screenshots/...`）
4. `npm run dev` でローカル確認
5. CLS（Cumulative Layout Shift）がないか確認
6. 画像の読み込み速度を確認（Lighthouse）
